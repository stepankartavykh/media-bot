Техническое задание (Начало проекта)

Бот - это лишь frontend-часть огромной системы (потенциальной) сборки, анализа, валидации информации и формирования "объективной" повестки у аудитории.



1 Начальные команды для настройки окружения у пользователя
1.1 /start Начало диалога. При повторном вызове стираются настройки для пользователя и конфигурация настраивается заново.
1.2 /help - справка по командам
1.3 /last - последние новости
1.4 /add_source - добавить желаемый источник информации
1.5 /observe - начать мониторинг ресурсов

1) Пользователь заходит в бот
2) На выбор получает несколько команд (команда по отслеживанию ресурсов, ввод )
3) На вход системе пользователь отдает список тем, в которых он заинтересован.

Система кэша должна в каждый момент времени выдавать актуальную "компиляцию" информации.
Должны скрываться новости по мере поступления новых "экстренных" новостей.

- Каков вид одного сообщения?
- Каким образом ранжировать новости по смыслу, эмоциональной окраске, упоминаниям в других источниках?
- Каким образом наиболее понятным способом предоставить информацию пользователю (размер сообщения и остальные характеристики)?




Вопросы:
1) Система кэширования - как должна работать?
2) Каким образом определить список тем по которым будем в кеше хранить выборку тем?
